# 图文内容创作：从视频创作中借鉴的方法论

此笔记梳理了如何将视频创作系统（如 OpenMontage 体系）中的影视级创作方法（如创意需求理解、镜头感、冲突设计、质量审核）迁移并应用到传统的图文内容创作中，形成一套结构化的内容生产工作流。

---

## 1. 核心迁移思路：内容承诺与结构冲突

## 1. 先确定“内容承诺”

图文创作开始前，先明确读者最终能得到什么，而不是先想标题或配图。

例如：

* 看完能理解一个概念
* 看完能完成一个操作
* 看完能做出选择
* 看完能改变一个错误认知
* 看完能获得情绪共鸣

可以写成一句话：

> 这篇内容帮助谁，在什么场景下，解决什么问题。

这相当于视频项目里的“创意需求理解”和“交付承诺”。

---

## 2. 用冲突组织内容，不要只是罗列知识

图文最常见的问题是：

```text
第一点……
第二点……
第三点……
```

信息可能正确，但读起来没有推动力。

可以借鉴项目里的 “But–Therefore”：

```text
大家通常认为 A，
但 A 无法解释 B，
因此需要引入 C，
可是 C 又会带来 D，
所以真正有效的方法是 E。
```

适合知识文章、公众号、小红书长图、产品介绍和案例分析。

例如写 AI 提示词：

```text
很多人认为提示词越长越好。

但模型失败往往不是因为描述不够多，
而是因为主体、动作和空间关系不清楚。

因此，比堆风格词更重要的是先把画面结构写明白。
```

这样比直接列“提示词五原则”更容易让人读下去。

---

## 3. 用“误解—失败—正确解释”的结构

这是图文科普特别好用的结构：

1. 读者通常怎么理解
2. 为什么这种理解看似合理
3. 它在什么情况下失效
4. 正确的机制是什么
5. 应该如何实践

例如：

```text
误解：AI 生图的关键是多写高级风格词。

为什么看起来合理：
风格词确实会直接影响画面外观。

问题：
风格词不能解决人物动作混乱、空间关系错误和主体漂移。

正确方法：
先描述主体、动作、场景和构图，再补充风格。
```

这个结构同时提供了钩子、认知冲突和解决方案。

---

## 4. 把抽象形容词改成可视化描述

项目里非常值得借鉴的一点是：不要停留在“高级、治愈、科技感、氛围感”这些词上。

图文创作也一样。

例如不要只写：

> 做一张有高级感的封面。

可以改成：

> 大面积留白，主体只占画面三分之一；使用低饱和背景；标题不超过两行；只保留一种强调色；光线柔和、阴影边缘清晰。

不要只写：

> 文案要有力量。

可以改成：

> 使用短句；先陈述困境，再给出确定判断；避免连续解释；结尾使用一句可以单独截图传播的结论。

也就是把主观感受拆成可执行参数。

---

## 5. 图不是装饰，而是承担信息任务

项目里每个镜头都要服务脚本。放到图文中，就是每张图都应该有明确作用。

图片大致可以承担这些角色：

| 图片作用   | 示例          |
| ------ | ----------- |
| 吸引注意   | 封面主视觉、冲突场景  |
| 解释概念   | 示意图、结构图     |
| 提供证据   | 截图、数据图、案例对比 |
| 降低理解难度 | 步骤图、流程图     |
| 营造情绪   | 场景图、人物图     |
| 控制阅读节奏 | 章节过渡图、大字卡   |
| 强化记忆   | 总结图、金句图     |

如果一张图删掉以后，文章信息和阅读体验几乎没有变化，那它可能只是装饰。

---

## 6. 先做“视觉脚本”，再生成图片

不要写完文章后再临时问：“这里放什么图？”

可以先做一个简单表格：

| 段落  | 核心信息 | 图片功能   | 图片形式     |
| --- | ---- | ------ | -------- |
| 开头  | 常见误区 | 制造冲突   | 错误与正确对比图 |
| 第二段 | 解释原因 | 降低理解难度 | 结构示意图    |
| 第三段 | 给出方法 | 指导操作   | 4 步流程图   |
| 案例  | 展示结果 | 提供证据   | 前后对比     |
| 结尾  | 总结记忆 | 强化传播   | 金句卡片     |

这相当于视频里的分镜和 B-roll 规划。

---

## 7. 图像提示词要包含结构，而不仅是风格

借鉴项目的视频提示词框架，图像提示词可以写成：

```text
主体 + 动作或状态 + 环境 + 构图 + 光线 + 风格 + 用途
```

例如，不要只写：

> 一个充满科技感的 AI 创作者。

可以写：

> 一名年轻创作者坐在桌前编辑内容，桌面有笔记本电脑和手写草稿，背景为简洁工作室；人物位于画面右侧，左侧留出标题空间；冷色环境光配暖色桌灯；写实 editorial photography 风格；用于公众号横版头图，不出现任何文字。

这里尤其值得保留的是：

* 明确主体位置
* 明确留白位置
* 明确图片用途
* 明确画幅
* 明确不要生成文字

---

## 8. 不要依赖 AI 生成准确文字

OpenMontage 明确建议：图片里的标题、字幕和标签应在后期程序化叠加。

图文创作同样适用。

更稳妥的流程是：

1. AI 只生成无字底图
2. 在 Canva、Figma、Photoshop 或排版工具里加文字
3. 统一字体、字号、颜色和边距
4. 检查手机端可读性

原因包括：

* AI 生成文字容易乱码
* 字体风格不统一
* 标题难以修改
* 不利于形成品牌模板
* 多张图难以保持排版一致

---

## 9. 建立一份视觉风格手册

为了避免每张图像来自不同世界，可以提前固定：

* 主色和辅助色
* 背景色
* 字体
* 圆角和边框
* 插画或摄影风格
* 光线类型
* 构图习惯
* 图标风格
* 图片比例
* 人物特征
* 禁止出现的元素

例如：

```text
主色：深蓝、米白
强调色：橙色
构图：大量留白，主体偏右
插画：扁平几何，不使用复杂渐变
摄影：柔和自然光，低饱和
文字：标题粗体，正文常规体
封面：最多一个主体，不放小字
```

这对应项目里的视觉系统、Hero Reference 和一致性锚点。

---

## 10. 先做一张“主参考图”

如果是一组图文、系列文章或品牌账号，可以先生成或设计一张最满意的主视觉。

这张图用来确定：

* 人物样貌
* 色彩
* 材质
* 光影
* 构图
* 整体气质

后续图片都参考它，而不是每次从零写提示词。

这对以下场景尤其有效：

* 小红书系列图
* 连载科普
* 品牌公众号
* 儿童故事
* 漫画式教程
* 角色型 IP 内容

---

## 11. 真实内容与生成内容分开选择

可以直接借鉴项目的素材决策逻辑。

### 更适合真实图片或截图

* 产品界面
* 操作教程
* 新闻事件
* 数据证据
* 人物采访
* 真实案例
* 商品效果
* 实际环境

### 更适合 AI 生成

* 抽象概念
* 视觉隐喻
* 不存在的场景
* 品牌插画
* 情绪氛围
* 难以拍摄的历史或未来场景
* 风格统一的配图

一个简单原则：

> 需要可信度时用真实素材，需要想象力和定制性时用生成素材。

不要用 AI 假图代替本该展示的真实证据。

---

## 12. 用不同视觉形式表达不同知识

并不是所有内容都适合生成插画。

| 信息类型 | 推荐形式    |
| ---- | ------- |
| 流程   | 流程图     |
| 对比   | 双栏对照    |
| 时间变化 | 时间轴     |
| 数据趋势 | 折线图、柱状图 |
| 层级关系 | 树状图     |
| 因果关系 | 箭头图     |
| 抽象概念 | 视觉隐喻    |
| 操作步骤 | 截图加标注   |
| 观点总结 | 大字卡     |
| 真实案例 | 实拍图或截图  |

项目里“确定性视觉用程序生成，开放式视觉用模型生成”的原则，在图文里也非常重要。

图表、文字、流程图最好用排版工具完成；AI 生图更适合插画、场景和概念视觉。

---

## 13. 控制信息密度和阅读节奏

视频要控制镜头节奏，图文要控制阅读节奏。

不要每一屏都塞满信息，可以交替使用：

```text
强钩子
→ 短段解释
→ 一张图
→ 核心结论
→ 案例
→ 留白
→ 下一部分
```

尤其是手机阅读场景：

* 一段尽量只讲一个意思
* 重要结论单独成段
* 长段落拆成小节
* 每隔一段安排视觉变化
* 图片里的文字不要过小
* 不要连续放很多风格相同的图

图文版的“幻灯片风险”就是：每一页都是模板化大字加背景图，视觉变化很多，但没有真正的信息推进。

---

## 14. 建立图文审核清单

发布前可以检查四个层面。

### 内容

* 开头是否迅速提出问题
* 是否存在明确主线
* 每一节是否推动结论
* 是否有重复表达
* 结论是否兑现标题承诺

### 图片

* 每张图是否有作用
* 图片和正文是否一致
* 人物、色彩和风格是否统一
* 是否出现乱码、畸形手指或逻辑错误
* 真实内容是否被误画成生成内容

### 排版

* 手机端是否易读
* 标题层级是否清楚
* 字号是否统一
* 留白是否足够
* 图片比例是否一致

### 可信度

* 数据是否有来源
* 截图是否真实
* 案例是否可验证
* AI 图片是否可能让人误认为真实记录
* 是否存在过度承诺

---

## 一套可直接套用的图文创作流程

```text
1. 明确读者和内容承诺
2. 找出核心冲突或常见误解
3. 搭建文章叙事结构
4. 为每一段定义图片任务
5. 选择真实素材、图表或 AI 图片
6. 建立视觉风格和主参考图
7. 批量生成无字图片
8. 统一排版和文字叠加
9. 检查事实、视觉一致性和阅读节奏
10. 输出封面、正文图和总结图
```

最核心的借鉴可以归纳成一句话：

> 先把内容、视觉和阅读节奏结构化，再让 AI 参与生成；不要让 AI 同时决定写什么、画什么以及如何排版。
